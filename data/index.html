    ▸ Bootloader & mode controls — direct state transitions
    

      

        Boot app0/app1 = flip OTADATA pointer (нужен DFU). Enter stub = CRSF 'bl' frame (нужен app). Exit DFU = RUN_USER_CODE (нужен DFU/stub).
      

      

        
        
      

      

        
        
      

      

        ⓘ What can't the plate do? (architectural limits)
        

          • Force WiFi programmatic — vanilla не диспатчит UART-MSP. 3× BOOT-press на RX, 60s-auto-wifi, или через handset.

          • Exit WiFi via CRSF — нет opcode. HTTP POST на 10.0.0.1/reboot с телефона или power-cycle.

          • Force ROM DFU — только BOOT-провод (GPIO3) без RESET. Физ. BOOT+power-cycle.

      Единственный сигнальный разъём платы (GP10/GP11 + 5V/GND)
      используется для всех фич: батарея, приёмник, сервопривод/мотор. Одновременно работает только один режим.
      Выбранный режим сохраняется между перезагрузками.

USB Mode

    

      USB-дескриптор фиксируется при загрузке — любое переключение требует перезагрузки.

      • CDC — стандартный USB Serial (прошивка, отладка).

      • USB2TTL — CDC + прозрачный мост на аппаратный UART1.

      • USB2I2C — эмулятор CP2112 HID (DJI Battery Killer, bqStudio, SMBus-инструменты).
